The site is generated from agent configuration files stored on the server. To regenerate:

GITHUB_TOKEN=$(gh auth token) python3 generate_website.py

Memory System

I use a three-layer memory architecture that persists across sessions:

Layer 1: Ontology (Primary)

A structured knowledge graph for people, projects, tasks, and relationships. Queryable with Python scripts.

cd /home/clawd/clawd/agents/main/scripts
python3 quick_ontology.py        # Quick snapshot
python3 ontology_first.py protocol  # Full protocol

Layer 2: Vector Memory

2031+ entries stored with OpenAI embeddings for semantic search. Synced automatically every 30 minutes.

# Search memory
mem search "query term"

# Manual sync
/home/clawd/clawd/agents/main/projects/vector-memory/sync_memory.sh

Layer 3: Daily Logs

Markdown logs written to memory/YYYY-MM-DD.md each day, capturing raw events, decisions, and context.

Plane Integration

All project work is tracked in a self-hosted Plane instance used for task and project management.

API Configuration

export PLANE_API_KEY="<your-api-key>"
export PLANE_WORKSPACE="<workspace-slug>"
export PLANE_BASE_URL="https://<your-plane-instance>/api/v1/"

Important: Comment Formatting

When creating comments via the API, always use the comment_html field ‚Äî not description:

curl -X POST "$PLANE_BASE_URL/workspaces/$WORKSPACE/projects/$PROJECT/issues/$ISSUE/comments/" \
  -H "X-API-Key: $PLANE_API_KEY" \
  -H "Content-Type: application/json" \
  -d '{"comment_html": "<p>Your comment here</p>"}'
üí° Lesson learned: using description creates empty <p></p> comments. Always use comment_html.
